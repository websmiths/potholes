# Petition signatures

This directory is the canonical record of petition signatures. Each `.md`
file is one signature. Files are produced by `node build.mjs --import` from
a Formspree CSV export — never edited by hand.

## File format

```markdown
---
id: ab12cd34                         # short id, derived from email hash
submittedAt: 2026-05-12T14:33:22+10:00
role: resident                       # resident | local | commuter | business | other
suburb: Rosebank 2480                # optional, free text from signer
emailHash: 7f8a2c…                   # SHA-256 of normalized email; plain email is NEVER committed
displayName: Anonymous Fox Road resident
displayPublicly: false               # signer opted in to public name display
storyPublic: false                   # signer opted in to public story display
---

<!-- The body is included only if storyPublic is true -->
```

## Privacy contract

- The signer's plain email **never appears anywhere in this repo**. The hash
  is only used for deduplication and a soft client-side "you've already
  signed" hint.
- Names appear publicly **only if** the signer ticked the "show my name on
  the public list" box at signing time.
- Story bodies appear publicly **only if** the signer ticked "share my story
  publicly".
- Suburbs and roles are aggregated for stats but otherwise non-identifying.
- Withdrawing a signature: delete the corresponding `.md` file (or all files
  matching a known email hash) and re-run `node build.mjs --stats`.

## Stats regeneration

`stats.json` and `signatures-hashes.json` at the repo root are derived
artefacts. They are regenerated by `node build.mjs --stats` (or as a side
effect of `--import`). Do not edit them by hand.
